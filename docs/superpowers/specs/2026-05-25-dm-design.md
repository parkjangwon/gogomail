# DM — 인스턴트 메시징 설계

**Goal:** 같은 도메인 사용자 간 1:1 DM과 그룹 채팅방을 제공하는 경량 메신저 기능

**Architecture:** REST + 클라이언트 폴링(polling). WebSocket·SSE 없음. 기존 GoGoMail
webmail 사이드바에 글로벌 패널로 노출. 메시지는 per-room AES-256-GCM으로 암호화,
방이 닫히면 키를 먼저 파기해 영구 복호화 불가 상태로 만든다.

**Tech Stack:** Go (backend), PostgreSQL (메시지 영속), MinIO (첨부파일 스토리지),
AES-256-GCM (메시지 암호화), Next.js (frontend panel)

---

## 1. 범위 및 원칙

- **이름:** DM (언어 무관, 다국어 처리 불필요)
- **대상:** 같은 도메인(`domain_id`) 사용자 간 전용. 외부 도메인 불가.
- **방 유형:** 1:1 다이렉트(`direct`) / 그룹(`group`) — 생성 시 유형 고정
- **1:1 방:** 멤버 추가 불가. 같은 두 사용자 사이에 방은 항상 하나만 존재(중복 생성 차단)
- **그룹 방:** 처음부터 그룹으로 생성. 공개(`public`) / 비공개(`private`) 속성 보유.
  방장(`owner`)이 멤버 초대·강퇴·위임 권한 보유
- **프라이버시 원칙:** 채팅 내역은 참여자만 열람 가능. 시스템 관리자 포함 어떠한 예외도 없음.
  방이 닫히면 키를 먼저 파기하여 시스템적으로 복구 불가능한 상태를 만든다.
- **제외 기능:** 타이핑 표시, 리치 에디터, 온라인 프레젠스, 스레드/답글

## 2. 데이터 모델

### dm_rooms
```sql
id            UUID PRIMARY KEY
company_id    UUID NOT NULL REFERENCES companies(id)
domain_id     UUID NOT NULL REFERENCES domains(id)
room_type     TEXT NOT NULL CHECK (room_type IN ('direct','group'))
visibility    TEXT CHECK (visibility IN ('public','private'))
              -- 그룹만 사용. direct는 NULL. 기본값 'private'
name          TEXT            -- 그룹만. direct는 NULL
owner_id      UUID REFERENCES users(id)  -- 그룹만. direct는 NULL
created_by    UUID NOT NULL REFERENCES users(id)
created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
```

### dm_room_keys
```sql
-- 방마다 고유한 AES-256-GCM 대칭 키. 앱 마스터 키로 한 번 더 암호화해 저장.
-- 이 행이 삭제되면 해당 방의 모든 메시지는 영구적으로 복호화 불가.
room_id        UUID PRIMARY KEY REFERENCES dm_rooms(id) ON DELETE CASCADE
key_ciphertext BYTEA NOT NULL   -- AES-256-GCM(room_key, master_key)
created_at     TIMESTAMPTZ NOT NULL DEFAULT now()
```

### dm_participants
```sql
id                    UUID PRIMARY KEY
room_id               UUID NOT NULL REFERENCES dm_rooms(id) ON DELETE CASCADE
user_id               UUID NOT NULL REFERENCES users(id)
joined_at             TIMESTAMPTZ NOT NULL DEFAULT now()
last_read_message_id  UUID        -- 읽음 계산 기준
UNIQUE (room_id, user_id)
```

### dm_messages
```sql
-- body, attachment_storage_path 는 해당 방의 room_key 로 AES-256-GCM 암호화된 값.
-- DB를 직접 조회해도 평문을 알 수 없다.
id                        UUID PRIMARY KEY
room_id                   UUID NOT NULL REFERENCES dm_rooms(id)
sender_id                 UUID REFERENCES users(id)  -- 시스템 메시지는 NULL
message_type              TEXT NOT NULL CHECK (message_type IN ('text','file','drive_link','system'))
body                      BYTEA NOT NULL              -- AES-256-GCM ciphertext
attachment_storage_path   BYTEA    -- MinIO 경로를 암호화한 값 (nullable)
attachment_name           TEXT     -- 파일명은 평문 (검색용)
attachment_size           BIGINT   -- bytes
attachment_mime_type      TEXT
drive_file_id             UUID REFERENCES drive_nodes(id)
created_at                TIMESTAMPTZ NOT NULL DEFAULT now()
edited_at                 TIMESTAMPTZ  -- 수정 시 갱신 (본인만 가능)
deleted_at                TIMESTAMPTZ  -- 소프트 삭제 (본인만 가능)

INDEX (room_id, created_at DESC)
INDEX (room_id, id DESC)
```

### dm_reactions
```sql
id          UUID PRIMARY KEY
message_id  UUID NOT NULL REFERENCES dm_messages(id) ON DELETE CASCADE
user_id     UUID NOT NULL REFERENCES users(id)
emoji       TEXT NOT NULL  -- 유니코드 단일 이모지 (평문. 반응 종류는 비밀 아님)
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
UNIQUE (message_id, user_id, emoji)

INDEX (message_id)
```

### dm_message_urls
```sql
-- 메시지 body에서 전송 시점에 추출한 URL. 평문 저장 → 빠른 링크 모아보기.
id          UUID PRIMARY KEY
message_id  UUID NOT NULL REFERENCES dm_messages(id) ON DELETE CASCADE
room_id     UUID NOT NULL REFERENCES dm_rooms(id) ON DELETE CASCADE
url         TEXT NOT NULL
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()

INDEX (room_id, created_at DESC)
```

### dm_invites
```sql
-- 비공개 그룹 방 초대 링크. 1회 사용 후 효력 소멸.
id          UUID PRIMARY KEY  -- 이 UUID 자체가 초대 토큰
room_id     UUID NOT NULL REFERENCES dm_rooms(id) ON DELETE CASCADE
created_by  UUID NOT NULL REFERENCES users(id)
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
expires_at  TIMESTAMPTZ NOT NULL  -- 생성 후 7일
used_at     TIMESTAMPTZ           -- NULL = 미사용. 값이 있으면 소진
used_by     UUID REFERENCES users(id)

INDEX (room_id)
```

## 3. 암호화 설계

### 마스터 키
- 앱 시작 시 환경변수(`GOGOMAIL_DM_MASTER_KEY`) 또는 KMS에서 로드
- DB에 저장하지 않음
- 마스터 키 없이는 room_key 복호화 불가 → 메시지 복호화 불가

### Room Key 생명주기
```
방 생성 → crypto/rand 32바이트 room_key 생성
         → AES-256-GCM(room_key, master_key) → dm_room_keys 저장

메시지 전송 → dm_room_keys에서 key_ciphertext 조회
             → master_key로 복호화 → room_key 획득
             → AES-256-GCM(body, room_key) → dm_messages 저장

방 종료 (참여자 0명) →
  1. dm_room_keys 행 즉시 삭제 (키 파기, 이후 복호화 불가)
  2. dm_messages 하드 삭제
  3. dm_participants 하드 삭제
  4. dm_invites 하드 삭제
  5. dm_rooms 하드 삭제
```

### 관리자 차단
- `internal/dm` 패키지만 복호화 로직 보유
- 관리자 API(`admin.go`)는 DM 관련 엔드포인트 없음
- DM 메시지 조회는 반드시 참여자 JWT 검증 후에만 복호화 수행
- 감사 로그에 메시지 내용 절대 기록 안 함 (방 ID, 이벤트 유형만 기록)

## 4. API

### 방 관리
```
GET    /api/v1/dm/rooms
  → 내가 참여 중인 방 목록 + unread_count 포함

GET    /api/v1/dm/rooms/public
  → 도메인 내 공개 그룹 목록 (미참여 방 포함, 이름·인원수만 반환)

POST   /api/v1/dm/rooms
  body: { room_type: 'direct'|'group', user_ids: [...], name?: string,
          visibility?: 'public'|'private' }
  → direct: 이미 방 있으면 기존 방 반환 (중복 차단)
  → group: 즉시 생성, 생성자가 owner. visibility 기본값 'private'

POST   /api/v1/dm/rooms/:id/members
  body: { user_ids: [...] }
  → 그룹 방장만. 같은 도메인 유저만 초대 가능

DELETE /api/v1/dm/rooms/:id/members/:user_id
  → 방장: 타인 강퇴 / 본인: 자발적 퇴장
  → 퇴장 시 시스템 메시지 자동 삽입
  → 참여자가 0명이 되면 방 전체 하드 삭제 (키 파기 포함)

PATCH  /api/v1/dm/rooms/:id/owner
  body: { user_id: uuid }
  → 현재 방장만 호출 가능. 대상 유저가 참여자여야 함
  → 시스템 메시지 삽입 후 owner_id 변경
```

### 초대 링크 (비공개 그룹)
```
POST   /api/v1/dm/rooms/:id/invites
  → 방장만 생성 가능. 7일 유효. 단일 사용.
  → 응답: { invite_url: "https://.../dm/join/{uuid}" }

POST   /api/v1/dm/join/:token
  → 토큰 유효성 검증 (미사용 + 미만료 + 같은 도메인)
  → 즉시 used_at 갱신 (이후 토큰 무효화)
  → 참여자로 등록 + 시스템 메시지 삽입
  → 이미 참여 중이면 409
```

### 메시지
```
GET  /api/v1/dm/rooms/:id/messages?before=<message_id>&limit=50
  → 히스토리 (무한 스크롤, 오래된 순 정렬). 복호화 후 반환.

GET  /api/v1/dm/rooms/:id/messages?after=<message_id>
  → 폴링 엔드포인트. 새 메시지만 반환 (빈 배열도 정상)

POST /api/v1/dm/rooms/:id/messages
  body: { body: string, drive_file_id?: uuid }
  → 전송 시 body에서 URL 파싱 → dm_message_urls에 평문 저장 (메시지 저장과 동일 트랜잭션)

PATCH  /api/v1/dm/messages/:id
  body: { body: string }
  → 본인 text 메시지만. edited_at 갱신.

DELETE /api/v1/dm/messages/:id
  → 본인 메시지만. 소프트 삭제.
```

### 첨부파일 업로드
```
POST /api/v1/dm/rooms/:id/attachments
  Content-Type: multipart/form-data, 최대 20 MB
  → MinIO 저장 후 경로를 room_key로 암호화해 dm_messages에 저장
  → Drive에 노출 안 됨
  → 다운로드: presigned URL (TTL 1시간) 발급
```

### 이모지 리액션
```
PUT /api/v1/dm/messages/:id/reactions/:emoji
  → 없으면 추가, 있으면 제거 (토글)
```

### 읽음 처리
```
POST /api/v1/dm/rooms/:id/read
  body: { last_message_id: uuid }
  → dm_participants.last_read_message_id 갱신
```

### 대화방 내 검색
```
GET /api/v1/dm/rooms/:id/search?q=<text>&before=<message_id>&limit=20
  → 참여자 인증 후 room_key 복호화 → 배치 복호화 → 대소문자 무관 부분 문자열 매칭
  → 최근 메시지부터 역순 탐색. 결과에 전후 컨텍스트 포함.
  → 인덱스 없는 선형 스캔이므로 탐색 범위를 최근 10,000건으로 제한
  → 시스템 메시지·삭제 메시지 제외
```

### 미디어·링크 모아보기
```
GET /api/v1/dm/rooms/:id/media?type=files&before=<message_id>&limit=30
  → attachment_name, attachment_size, attachment_mime_type (평문) 조회
  → 이미지·동영상·문서별 필터: ?type=image|video|file
  → presigned URL 포함 반환

GET /api/v1/dm/rooms/:id/media?type=links&before=<message_id>&limit=30
  → dm_message_urls 평문 테이블 조회 (복호화 불필요)
  → 전송 시간·발신자 포함 반환

GET /api/v1/dm/rooms/:id/media?type=drive&before=<message_id>&limit=30
  → drive_file_id 컬럼 (평문) 조회 → Drive 파일 메타데이터 join
```

## 7. 시스템 메시지

`message_type = 'system'`, `sender_id = NULL`. 삭제·리액션 불가.
시스템 메시지 body도 room_key로 암호화.

| 이벤트 | body |
|---|---|
| 멤버 퇴장 (자발/강퇴) | "{display_name}님이 나갔습니다." |
| 멤버 초대 (방장 직접) | "{display_name}님이 초대되었습니다." |
| 멤버 입장 (초대 링크) | "{display_name}님이 참여했습니다." |
| 방장 위임 | "방장이 {display_name}님에게 권한을 위임했습니다." |

## 8. UI 구조

### 진입점 (좌측 사이드바 하단)

```
│  (메일 목록 등 메인 영역)  │
├───────────────────────────┤
│  💬  DM          ← 추가   │
│  🔔  알림센터              │
│  ⚙️  설정                  │
└───────────────────────────┘
```

- DM 아이콘에 unread 뱃지
- 글로벌 단축키 `D` 로 패널 토글 (충돌 시 `` ` `` 사용)

### DM 패널 레이아웃

```
┌─────────────────────────────────────────────────┐
│ DM                                 [새 방 만들기] ✕ │
├────────────────┬────────────────────────────────┤
│ 방 목록         │ 김철수  오늘 12:31             │
│                │ 안녕하세요 PR 확인해주세요       │
│ 👤 김철수  ●2  │                                │
│ 👥 백엔드팀     │ 나  12:33                      │
│ 👤 이영희       │  네 확인했습니다               │
│                │                    👍 1  읽음   │
│                ├────────────────────────────────┤
│                │ [📎] [🗂]  메시지 입력...  [전송] │
└────────────────┴────────────────────────────────┘
```

- **좌**: 방 목록. unread count 뱃지. 방 이름(그룹) 또는 상대방 이름(1:1)
- **우**: 대화 뷰. 내 메시지 오른쪽, 상대 메시지 왼쪽 (Instagram DM 스타일)
- **📎**: 직접 파일 업로드 (20 MB)
- **🗂**: Drive 파일 선택
- 이모지 리액션: 메시지 호버 시 이모지 팝오버

### 메시지 버블

- 평문 텍스트. URL은 자동 링크
- 파일: 아이콘 + 파일명 + 용량. 클릭 시 presigned URL 다운로드
- Drive 파일: 파일명 + Drive 아이콘. 클릭 시 Drive 뷰어
- 수정된 메시지: 본문 뒤 "(수정됨)" 회색 소문자
- 삭제된 메시지: "삭제된 메시지입니다." 회색 이탤릭체
- 시스템 메시지: 중앙 정렬, 회색 소문자

## 10. 보안 및 제한

- **암호화:** 모든 메시지 body·첨부파일 경로는 per-room AES-256-GCM으로 암호화
- **키 파기:** 참여자 0명 → room_key 즉시 삭제 → 시스템적 복호화 불가
- **관리자 차단:** admin API에 DM 열람 경로 없음. 감사 로그에 메시지 내용 미기록
- **방 접근:** 참여자가 아닌 요청 → 404 (방 존재 노출 방지)
- **도메인 격리:** 모든 쿼리에 `domain_id` 필터
- **초대 링크:** 1회 사용 후 즉시 무효화. 7일 만료. 같은 도메인 유저만 사용 가능
- **공개 방 탐색:** 이름·인원수만 노출. 메시지 미노출
- **방장 강퇴 방지:** 방장은 본인 강퇴 불가. 위임 후에만 퇴장 가능
- **첨부파일:** MIME 검증, 20 MB hard limit, presigned URL (TTL 1시간)
- **이모지 유효성:** 서버 측 유니코드 이모지 검증
