# DM — 인스턴트 메시징 설계

**Goal:** 같은 도메인 사용자 간 1:1 DM과 그룹 채팅방을 제공하는 경량 메신저 기능

**Architecture:** REST + 클라이언트 폴링(polling). WebSocket·SSE 없음. 기존 GoGoMail
webmail 사이드바에 글로벌 패널로 노출.

**Tech Stack:** Go (backend), PostgreSQL (메시지 영속), MinIO (첨부파일 스토리지),
Next.js (frontend panel)

---

## 1. 범위 및 원칙

- **이름:** DM (언어 무관, 다국어 처리 불필요)
- **대상:** 같은 도메인(`domain_id`) 사용자 간 전용. 외부 도메인 불가.
- **방 유형:** 1:1 다이렉트(`direct`) / 그룹(`group`) — 생성 시 유형 고정
- **1:1 방:** 멤버 추가 불가. 같은 두 사용자 사이에 방은 항상 하나만 존재(중복 생성 차단)
- **그룹 방:** 처음부터 그룹으로 생성. 방장(`owner`)이 멤버 초대·강퇴 권한 보유
- **제외 기능:** 타이핑 표시, 리치 에디터, 온라인 프레젠스, 스레드/답글

## 2. 데이터 모델

### dm_rooms
```sql
id            UUID PRIMARY KEY
company_id    UUID NOT NULL REFERENCES companies(id)
domain_id     UUID NOT NULL REFERENCES domains(id)
room_type     TEXT NOT NULL CHECK (room_type IN ('direct','group'))
name          TEXT            -- 그룹만. direct는 NULL
owner_id      UUID REFERENCES users(id)  -- 그룹만. direct는 NULL
created_by    UUID NOT NULL REFERENCES users(id)
created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
```

### dm_participants
```sql
id                    UUID PRIMARY KEY
room_id               UUID NOT NULL REFERENCES dm_rooms(id) ON DELETE CASCADE
user_id               UUID NOT NULL REFERENCES users(id)
joined_at             TIMESTAMPTZ NOT NULL DEFAULT now()
last_read_message_id  UUID REFERENCES dm_messages(id)  -- 읽음 계산 기준
UNIQUE (room_id, user_id)
```

### dm_messages
```sql
id                        UUID PRIMARY KEY
room_id                   UUID NOT NULL REFERENCES dm_rooms(id)
sender_id                 UUID REFERENCES users(id)  -- 시스템 메시지는 NULL
message_type              TEXT NOT NULL CHECK (message_type IN ('text','file','drive_link','system'))
body                      TEXT NOT NULL
attachment_storage_path   TEXT     -- MinIO 경로 (직접 업로드)
attachment_name           TEXT
attachment_size           BIGINT   -- bytes
attachment_mime_type      TEXT
drive_file_id             UUID REFERENCES drive_nodes(id)  -- Drive 링크
created_at                TIMESTAMPTZ NOT NULL DEFAULT now()
edited_at                 TIMESTAMPTZ  -- 수정 시 갱신 (본인만 가능)
deleted_at                TIMESTAMPTZ  -- 소프트 삭제 (본인만 가능)

INDEX (room_id, created_at DESC)
INDEX (room_id, id DESC)
```

### dm_reactions
```sql
id          UUID PRIMARY KEY
message_id  UUID NOT NULL REFERENCES dm_messages(id) ON DELETE CASCADE
user_id     UUID NOT NULL REFERENCES users(id)
emoji       TEXT NOT NULL  -- 유니코드 단일 이모지 (예: "👍", "✅", "❤️")
created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
UNIQUE (message_id, user_id, emoji)  -- 동일 이모지 중복 방지

INDEX (message_id)
```

## 3. API

### 방 관리
```
GET    /api/v1/dm/rooms
  → 내가 참여 중인 방 목록 + unread_count 포함

POST   /api/v1/dm/rooms
  body: { room_type: 'direct'|'group', user_ids: [...], name?: string }
  → direct: 이미 방 있으면 기존 방 반환 (중복 차단)
  → group: 즉시 생성, 생성자가 owner

POST   /api/v1/dm/rooms/:id/members
  body: { user_ids: [...] }
  → 그룹 방장만. 같은 도메인 유저만 초대 가능

DELETE /api/v1/dm/rooms/:id/members/:user_id
  → 방장: 타인 강퇴 / 본인: 자발적 퇴장
  → 퇴장 시 시스템 메시지 자동 삽입: "{display_name}님이 나갔습니다."
  → 별도 알림(push) 없음
```

### 메시지
```
GET  /api/v1/dm/rooms/:id/messages?before=<message_id>&limit=50
  → 히스토리 (무한 스크롤, 오래된 순 정렬)

GET  /api/v1/dm/rooms/:id/messages?after=<message_id>
  → 폴링 엔드포인트. 새 메시지만 반환 (빈 배열도 정상)

POST /api/v1/dm/rooms/:id/messages
  body: { body: string, drive_file_id?: uuid }
  → 텍스트 전송 또는 Drive 파일 링크 첨부

PATCH  /api/v1/dm/messages/:id
  body: { body: string }
  → 본인 메시지만. `text` 타입만 수정 가능 (파일·드라이브·시스템 메시지 불가)
  → edited_at 갱신

DELETE /api/v1/dm/messages/:id
  → 본인 메시지만. 소프트 삭제. body를 "삭제된 메시지입니다."로 대체 표시
```

### 첨부파일 업로드
```
POST /api/v1/dm/rooms/:id/attachments
  Content-Type: multipart/form-data
  제한: 20 MB
  → MinIO 경로: dm/{room_id}/{uuid}/{filename}
  → Drive에 노출되지 않음
  → 응답: { message_id: uuid }  (메시지가 즉시 생성됨)
```

### 이모지 리액션
```
PUT /api/v1/dm/messages/:id/reactions/:emoji
  → 없으면 추가, 있으면 제거 (토글)
  → emoji 파라미터: URL 인코딩된 유니코드 이모지
```

### 읽음 처리
```
POST /api/v1/dm/rooms/:id/read
  body: { last_message_id: uuid }
  → dm_participants.last_read_message_id 갱신
```

## 4. 읽음 표시 로직

### 1:1 다이렉트
- 내가 보낸 메시지 중 가장 최근 메시지 아래에 "읽음" 표시
- 조건: 상대방의 `last_read_message_id` ≥ 해당 메시지의 `created_at`

### 그룹
- 각 메시지 아래 "읽음 N명" (방 전체 인원 중 해당 메시지까지 읽은 수)
- 나 자신은 제외하고 계산

## 6. 시스템 메시지

`message_type = 'system'`, `sender_id = NULL`인 메시지. 생성 케이스:

| 이벤트 | body |
|---|---|
| 멤버 퇴장 (자발/강퇴) | "{display_name}님이 나갔습니다." |
| 멤버 초대 | "{display_name}님이 초대되었습니다." |

시스템 메시지는 삭제·리액션 불가.

## 7. UI 구조

### 진입점 (좌측 사이드바 하단)

```
│  (메일 목록 등 메인 영역)  │
├───────────────────────────┤
│  💬  DM          ← 추가   │
│  🔔  알림센터              │
│  ⚙️  설정                  │
└───────────────────────────┘
```

- DM 아이콘에 unread 뱃지 (읽지 않은 총 메시지 수)
- 글로벌 단축키 `D`로 패널 토글 (충돌 시 `` ` `` 사용)

### DM 패널 레이아웃

```
┌─────────────────────────────────────────────────┐
│ DM                                 [새 방 만들기] ✕ │
├────────────────┬────────────────────────────────┤
│ 방 목록         │ 김철수  오늘 12:31             │
│                │ 안녕하세요 PR 확인해주세요       │
│ 👤 김철수  ●2  │                                │
│ 👥 백엔드팀     │ 나  12:33                      │
│ 👤 이영희       │  네 확인했습니다               │
│                │                    👍 1  읽음   │
│                ├────────────────────────────────┤
│                │ [📎] [🗂]  메시지 입력...  [전송] │
└────────────────┴────────────────────────────────┘
```

- **좌**: 방 목록. unread count 뱃지. 방 이름(그룹) 또는 상대방 이름(1:1)
- **우**: 대화 뷰. 내 메시지 오른쪽 정렬, 상대 메시지 왼쪽 정렬 (Instagram DM 스타일)
- **📎**: 직접 파일 업로드 (20 MB 제한)
- **🗂**: Drive 파일 선택
- 이모지 리액션: 메시지 호버 시 이모지 팝오버 노출

### 메시지 버블

- 평문 텍스트. URL은 클릭 가능한 링크로 자동 변환
- 파일: 아이콘 + 파일명 + 용량 표시. 클릭 시 다운로드
- Drive 파일: 파일명 + Drive 아이콘. 클릭 시 Drive 뷰어
- 수정된 메시지: 본문 뒤에 "(수정됨)" 회색 소문자로 표시. `edited_at` 기준
- 삭제된 메시지: "삭제된 메시지입니다." 회색 이탤릭체
- 시스템 메시지: 중앙 정렬, 회색 소문자

## 9. 보안 및 제한

- 방 접근: 참여자가 아닌 사용자의 모든 요청 → 404 (방 존재 노출 방지)
- 도메인 격리: 모든 쿼리에 `domain_id` 필터 적용
- 첨부파일: MinIO presigned URL 발급 (직접 노출 금지), TTL 1시간
- 메시지 수정: 본인 `text` 메시지만. 수정 이력은 저장하지 않음 (최종본만 보존)
- 메시지 삭제: 본인 메시지만, 소프트 삭제
- 강퇴: 방장만 가능, 방장 본인 강퇴 불가
- 이모지 유효성: 서버 측 유니코드 이모지 검증 (악성 문자열 차단)
- 파일 업로드: MIME 타입 검증, 20 MB hard limit
